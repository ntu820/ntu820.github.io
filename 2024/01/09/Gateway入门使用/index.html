springcloud gateway入门

一：简介

Spring Cloud Gateway 是Spring Cloud的一个全新的API网关项目，目的是为了替换掉Zuul1，它基于Spring5.0 + SpringBoot2.0 + WebFlux（基于性能的Reactor模式响应式通信框架Netty，异步阻塞模型）等技术开发，性能于Zuul，官测试，Spring Cloud GateWay是Zuul1的1.6倍 ，旨在为微服务架构提供种简单有效的统的API路由管理式 。

1. 基本概念

路由（Route）是GateWay中最基本的组件之一，表示一个具体的路由信息载体，主要由下面几个部分组成：

  1. id：路由唯一标识，区别于其他的route
  2. url： 路由指向的目的地URL，客户端请求最终被转发到的微服务
  3. order： 用于多个Route之间的排序，数值越小越靠前，匹配优先级越高
  4. predicate：断言的作用是进行条件判断，只有断言为true，才执行路由
  5. filter: 过滤器用于修改请求和响应信息

当请求到达gateway后，会通过一些匹配条件，定位到真正的服务节点，并且在这个转发过程前后，进行一些细粒度的控制，其中 Predicate（断言） 是我们的匹配条件，Filter 是一个拦截器，有了这两点，再加上URL，就可以实现一个具体的路由。

核心思想：路由转发+执行过滤器链。

Route(路由) ：构建网关的基础模块，由ID、目标URL、过滤器等组成。

Predicate（断言） ：开发人员可以匹配HTTP请求中的内容（请求头和请求参数），如果请求断言匹配贼进行路由。

Filter（过滤） ：GateWayFilter的实例，使用过滤器，可以在请求被路由之前或者之后对请求进行修改。

三：核心思想

1. 内置路由断言工厂

作用：当请求到达gateway的时候，使用断言对请求进行匹配，如果匹配成功就路由转发，否则就返回404。

类型：内置断言工厂和自定义断言工厂

  • 基于Datatime的断言

    此类型断言根据时间进行判断，主要有三个：

    AfterRoutePredicateFactory：接收一个时间参数，判断请求时间是否晚于指定日期

    BeforeRoutePredicateFactory：接收一个时间参数，判断请求时间是否早于指定日期

    BetweenRoutePredicateFactory：接收两个时间参数，判断请求时间是否在指定时间段内

    spring:
  cloud:
    gateway:
      routes:
      - id: after_route
        uri: https://example.org
        predicates:
        - After=2017-01-20T17:42:47.789-07:00[America/Denver]

    可以通过ZoneDateTime.now()获取时间

  • 基于Cookie的断言

    判断cookie的key是否等于value，chocolate是否等于ch.p。支持正则表达式

    spring:
  cloud:
    gateway:
      routes:
      - id: cookie_route
        uri: https://example.org
        predicates:
        - Cookie=chocolate, ch.p

  • 基于请求头的断言

    判断请求头中某个key是否等于value，同样支持正则表达式

    spring:
  cloud:
    gateway:
      routes:
      - id: header_route
        uri: https://example.org
        predicates:
        - Header=X-Request-Id,\d+

  • 基于域名的断言

    多个域名用逗号分隔，支持多个域名，及通配符

    spring:
  cloud:
    gateway:
      routes:
      - id: host_route
        uri: https://example.org
        predicates:
        - Host=**.somehost.org,**.anotherhost.org

  • 基于请求方法的断言

    spring:
  cloud:
    gateway:
      routes:
      - id: method_route
        uri: https://example.org
        predicates:
        - Method=GET,POST

  • 基于路径的断言

    支持通配符，同样支持占位符。支持/red/1或者/red/blue或者/blue/green

    spring:
  cloud:
    gateway:
      routes:
      - id: path_route
        uri: https://example.org
        predicates:
        - Path=/red/{segment},/blue/{segment}

  • 基于查询参数的断言

    查询参数携带?green=xxx，同样支持正则表达式。如果要指定值，则使用Query=color,green

    spring:
  cloud:
    gateway:
      routes:
      - id: query_route
        uri: https://example.org
        predicates:
        - Query=green

  • 基于远端地址（IP）的断言

    spring:
  cloud:
    gateway:
      routes:
      - id: remoteaddr_route
        uri: https://example.org
        predicates:
        - RemoteAddr=192.168.1.1/24

  • 基于权重的断言

    发送10次请求，8次到weight_high，2次到weight_low。需要先配置组

    spring:
  cloud:
    gateway:
      routes:
      - id: weight_high
        uri: https://weighthigh.org
        predicates:
        - Weight=group1, 8
      - id: weight_low
        uri: https://weightlow.org
        predicates:
        - Weight=group1, 2

2. 自定义路由断言工厂

继承AbstractRoutePredicateFactory类，重新apply方法的逻辑。在apply方法中可以通过exchange.getRequet()拿到ServletHttpRequest对象，从而获取请求参数、请求方法、请求头等信息。

  1. 必须是spring组件，是一个bean
  2. 类名必须加上RoutePredicateFactory作为结尾
  3. 继承AbstractRoutePredicateFactory
  4. 声明静态内部类，声明属性来接收配置文件中对应的断言信息
  5. 结合shortcutFieldOrder进行绑定
  6. 通过apply进行逻辑判断，返回true匹配成功，返回false匹配失败

示例：实现Auth=zhangsan的断言


@Component
public class CheckRoutePredicateFactory extends AbstractRoutePredicateFactory<CheckRoutePredicateFactory.Config> {


    public CheckRoutePredicateFactory() {
        super(CheckRoutePredicateFactory.Config.class);
    }

    @Override
    public List<String> shortcutFieldOrder() {
        return Arrays.asList("name");
    }

    @Override
    public Predicate<ServerWebExchange> apply(CheckRoutePredicateFactory.Config config) {
        return new GatewayPredicate() {
            @Override
            public boolean test(ServerWebExchange exchange) {
                return config.getName().equals("zhangsan"); // 当这个name=zhangsan时进行断言
            }

        };
    }

    // 用于接收配置文件中断言信息
    @Validated
    public static class Config {
        private String name;

        public void setName(String name) {
            this.name = name;
        }

        public String getName() {
            return name;
        }
    }

}

3. 内置过滤器

路由过滤器允许以某种方式修改传入 HTTP 请求或传出 HTTP 响应。路由过滤器的范围限于特定路由。Spring Cloud Gateway 包含许多内置的 GatewayFilter 工厂。

4. 自定义过滤器

  1. 继承AbstractGatewayFilterFactory
  2. 类名必须以GatewayFilterFactory结尾

示例：实现check=lisi的过滤器

@Component
public class CheckGatewayFilterFactory extends AbstractGatewayFilterFactory<CheckGatewayFilterFactory.Config> {

    public CheckGatewayFilterFactory() {
        super(CheckGatewayFilterFactory.Config.class);
    }

    public List<String> shortcutFieldOrder() {
        return Arrays.asList("name");
    }

    public GatewayFilter apply(CheckGatewayFilterFactory.Config config) {
        return new GatewayFilter() {
            public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
                // 1. 获取name参数
                // 2. 如果！=lisi，正常
                // 3. 否则不正常
                String name = exchange.getRequest().getQueryParams().getFirst("name");
                if (!StringUtils.isEmpty(name)) {
                    if (config.getName().equals(name)) {
                        return chain.filter(exchange);
                    } else {
                        exchange.getResponse().setStatusCode(HttpStatus.valueOf("-1000"));
                        return exchange.getResponse().setComplete();
                    }
                }
                // 正常请求
                return chain.filter(exchange);
            }
        };
    }

    public static class Config {
        private String name;

        public String getName() {
            return name;
        }

        public void setName(String name) {
            this.name = name;
        }
    }
}

5. 内置全局过滤器

  1. 局部过滤器针对某个路由，全局过滤器针对所有路由
  2. 局部过滤器需要在路由中进行配置，全局过滤器一旦定义，就会投入使用

6. 自定义全局过滤器

实现GlobalFilter接口即可，一般授权、日志使用到。

@Component
public class LogFilter implements GlobalFilter {

    Logger log = LoggerFactory.getLogger(this.getClass());

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        log.info("正在使用全局过滤器进行记录，访问地址：" + exchange.getRequest().getPath().value() + "，远端IP：" + exchange.getRequest().getRemoteAddress().toString());
        return chain.filter(exchange);
    }
}

  • 使用yaml配置

示例：

function btnClick(){
	var xhr = new XMLHttpRequest(); // 创建XMLHttpRequest对象
	xhr.open('GET', 'http://localhost:8088/ocr/hello?name=lisi1'); // 设置请求的URL和HTTP方法为GET
	xhr.onreadystatechange = function() { // 定义状态改变时的处理函数

			console.log(responseText); // 输出结果到控制台
		}
	};
	xhr.send(); // 发送请求
	
}

配置跨域：

  • 使用配置类方式

    package com.hmifo.gateway.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.web.cors.CorsConfiguration;
import org.springframework.web.cors.reactive.CorsWebFilter;
import org.springframework.web.cors.reactive.UrlBasedCorsConfigurationSource;
import org.springframework.web.util.pattern.PathPatternParser;

@Configuration
public class CorsConfig {

    public CorsWebFilter corsFilter() {
        CorsConfiguration corsConfiguration = new CorsConfiguration();
        corsConfiguration.addAllowedMethod("*"); // 允许的方法
        corsConfiguration.addAllowedOrigin("*"); // 允许来源
        corsConfiguration.addAllowedHeader("*"); // 运行请求头参数

        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource(new PathPatternParser());
        source.registerCorsConfiguration("/**", corsConfiguration);

        return new CorsWebFilter(source);
    }
}
