养虾日记01-最全本地部署openclaw指南

一：前世今生

OpenClaw（曾用名：Clawdbot、Moltbot），一款可以部署在个人电脑上的AI代理，采用“龙虾”图标设计，slogan是“The AI that actually does things”，由程序员彼得·斯坦伯格开发。核心开发语言为TypeScript。

二：安装openclaw

环境选择：ubuntu2204桌面版

1. 前置条件

安装nodejs和git

  1. 安装nodejs，选择版本大于22即可
  2. 安装git，选择合适自己系统版本，ubuntu下直接使用 apt-get install git

2. 安装openclaw

  1. 在 Terminal中输入，最好以管理员模式

  2. 等待一段时间，注意，最好不要修改npm的默认仓库，可以进行 科学一下，保障安装顺利。

3. 配置openclaw

  1. 安装完成后会自动进入配置

  2. 去智普AI的开发者后台注册，创建API Key

    需要注意一点：openclaw对模型token的消耗是较大的，如果是调用厂商提供的API去使用，要注意token的消耗

  3. 配置channel通道，选择飞书。飞书最近出了官方的插件，支持流式输出，下期介绍

  4. 配置Skills，选择No，先不用着急配置，配置完成后可以通过UI界面进行Skill安装

  5. 配置Enable hooks，除了Skip for now，其他都选中

  6. 配置关于GOOGLE等相关信息，都选择NO

  7. 整个过程需要一点时间，等待即可。然后会默认打开浏览器，访问地址：http://127.0.0.1:18789/?token=xxxxxx，有的时候token不一定会有，可以从如下方式拿到。

至此openclaw配置完成。

4. 安全须知

尽管功能强大，OpenClaw 明确存在安全风险：

  1. 出现API 密钥泄露导致财产损失的案例
  2. 存在提示注入（prompt injection）等高危漏洞
  3. 在安装第一步就说明其需要强大的权限。“能删硬盘、能发邮件、能读聊天记录”的强权限特性，也引发对责任边界的讨论

最后

  • AI 并不完美，可能存在“幻觉”： 它有时会误解您的意图，或者生成看似合理但不准确的内容。
  • 部分操作不可逆转： 例如，AI发出后即成事实。
  • 应对建议： 对于涉及发送、修改、写入等重要操作，请务必做到“先预览，再确认”，切勿让 AI 处于完全脱离人工干预的“全自动驾驶”状态。
  • 先拿个人账号安全地“玩”起来，等后续安全隔离能力更成熟了，再考虑接入真实工作环境。
