使用kubeadm部署kubernetes-v1.19.8

使用kubeadm安装kubernetes

1.使用自己的kubeadm、kubelet、kubectl安装集群

安装kubernetes的版本为：v1.19.8，可参考：https://kuboard.cn/install/history-k8s/install-k8s-1.19.x.html，做了部分优化。

  1. 安装docker、kubelet，在所有机器上

    sh 0install_docker_kubelet.sh

  2. 在安装好docker和kubelet后，使用自己编译的kubeadm、kubelet、kubectl替换已经安装的文件

    scp kube-100.tar.gz root@x.x.x.x:/root
tar -zxvf kube-100.tar.gz
cd kube-100
cp kube* /usr/bin

  3. 导入镜像包

    scp k8s-v1.19.8-images.tar.gz root@x.x.x.x:/root
tar -zxvf k8s-v1.19.8-images.tar.gz
cd k8s-v1.19.8-images
# 带有_all标记的为master、node节点都需要导入
docker load < xxx.tar.gz

  4. 初始化master节点

    sh 1init_master.sh

  5. 添加工作节点

    # 只在 master 节点执行,
kubeadm token create --print-join-command
# 获取kubeadm join 命令及参数。在node节点输入上面得到的参数

  6. 配置命令自带补全

    yum install -y bash-completion

source /usr/share/bash-completion/bash_completion
echo 'source <(kubectl completion bash)' >>~/.bashrc
echo 'source <(kubectl completion bash)' >>~/.bashrc
source ~/.bashrc
kubectl completion bash >/etc/bash_completion.d/kubectl

  7. 查看集群过期时间

    kubeadm alpha certs check-expiration
    [check-expiration] Reading configuration from the cluster...

3.安装dashboard，部署和访问 Kubernetes 仪表板

参考：https://v1-22.docs.kubernetes.io/zh/docs/tasks/access-application-cluster/web-ui-dashboard，由于官方已经没有v1.19.8的文档了，这里参考v1.22的文档进行。

dashboard和kubernetes的对照关系表：https://github.com/kubernetes/dashboard/releases

  1. 生成自签证书

    通过官方的yaml文件来进行部署，这个方法存在很多问题。

    1. 该yaml文件的地址有时候并不能访问，需要科学；

    2. 该文件指定的dashboard的镜像也需要科学才能访问；

    3. 部署的dashboard的证书过期时间有问题，导致chrome、safari等都不能访问，仅firefox可以访问。

      所以需要对部署流程做调整，先创建自签证书，再用证书来部署。

    mkdir dashboard-cert && cd dashboard-cert
# 生成证书请求的key
openssl genrsa -out dashboard.key 2048
# 生成证书请求，<your_ip>换成自己的ip或域名
openssl req -new -out dashboard.csr -key dashboard.key -subj '/CN=<your_ip>'
# 生成自签证书，指定了过期时间36500天，默认365天
openssl x509 -days 36500 -req -in dashboard.csr -signkey dashboard.key -out dashboard.crt

  2. 根据生成的证书，创建密钥

    # 创建命名空间
kubectl create ns kubernetes-dashboard
# 创建密钥，"dashboard.key"、"dashboard.crt"是之前生成的自签证书的相关文件的路径。"kubernetes-dashboard-certs" 这个名称不要改
kubectl create secret generic kubernetes-dashboard-certs --from-file=dashboard.key --from-file=dashboard.crt -n kubernetes-dashboard
