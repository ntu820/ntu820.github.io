涛思2.6集群安装

涛思2.6集群安装

1.机器环境

节点/FQDN 角色 地址 备注
hmts1 mnode 172.11.219.121
hmts2 vnode 172.11.219.122
hmts3 vnode 172.11.219.123

2.准备安装目录

mkdir -p /home/data/taos/{data,core,soft,tmp,dump}
chmod 777 /home/data/taos/tmp

3.配置FQDN

echo '172.11.219.121  hmts1' >> /etc/hosts
echo '172.11.219.122  hmts2' >> /etc/hosts
echo '172.11.219.123  hmts3' >> /etc/hosts

4.配置NTP

ntpdate X.X.X.X

5.安装

5.1 配置taos.cnf

配置参数名称 含义 备注
firstEp 每个数据节点首次启动后连接的第一个数据节点
fqdn 必须配置为本数据节点的 FQDN
serverPort 配置本数据节点的端口号，缺省是 6030
arbitrator 副本数为偶数的时候，需要配置
numOfMnodes 系统中管理节点个数
mnodeEqualVnodeNum 一个 mnode 等同于 vnode 消耗的个数
offlineThreshold dnode 离线阈值，超过该时间将导致 Dnode 离线
statusInterval dnode 向 mnode 报告状态时长
arbitrator 系统中裁决器的 End Point
timezone 时区
balance 是否启动负载均衡
maxTablesPerVnode 每个 vnode 中能够创建的最大表个数
maxVgroupsPerDb 每个 DB 中能够使用的最大 vgroup 个数

一定要修改的参数是 firstEp 和 fqdn。在每个数据节点，firstEp 需全部配置成一样，但 fqdn 一定要配置成其所在数据节点的值。其他参数可不做任何修改，除非你很清楚为什么要修改。

整合：

  1. 创建涛思配置文件
firstEp                   hmts1:6030
fqdn                      hmts1
numOfThreadsPerCore       2.0
ratioOfQueryCores         2.0
numOfCommitThreads        4.0
numOfMnodes               3
dataDir                   /home/data/taos/data
logDir                    /home/data/taos/log
tempDir                   /home/data/taos/tmp
balance                   0
keepColumnName            1
timezone                  Asia/Shanghai (CST, +0800)
locale                    en_US.UTF-8
charset                   UTF-8
maxShellConns             100000
maxConnections            100000
rpcForceTcp               1
monitor                   1
logKeepDays               -1
debugFlag                 131
cDebugFlag                131
qDebugFlag                131
rpcDebugFlag              131
  1. 修改taosadapter.toml
[pool]
maxConnect = 128
maxIdle = 128

[log]
path = "/home/data/taos/log"

5.2 启动数据库[启动第一个数据节点]

# 启动
systemctl start taosd
# 检查服务状态 
systemctl status taosd

# 如果使用taosAdapter模块，启动命令如下
systemctl start taosadapter
# 检查服务状态
systemctl status taosadapter
# 设置开机自启动
systemctl enable taosadapter

启动集群

taos支持水平动态扩展，单个 taos 节点即可视为 1 个节点的 taos 集群。

  1. 启动第一个数据节点，见启动数据库。

    启动后续节点

    将后续的数据节点添加到现有集群，具体有以下几步：

    1. 按照上述方法，在其他节点安装 taos 服务端。

    2. 修改配置文件，taos.cnf。主要修改fqdn为当前节点的hostname，其他不变。

    3. 修改配置文件，taosadapter.toml。

    4. 配置完成后，分别在 2 和 3 节点启动 taosd 服务。

      # 启动
systemctl start taosd
# 检查服务状态 
systemctl status taosd

# 如果使用taosAdapter模块，启动命令如下
systemctl start taosadapter
# 检查服务状态
systemctl status taosadapter
# 设置开机自启动
systemctl enable taosadapter

    5. 在第一个数据节点，使用 CLI 程序 taos，登录进 TDengine 系统，执行命令：

      create dnode "hmts2:6030";
create dnode "hmts3:6030";

    6. 验证

      taos> show dnodes;
   id   |           end_point            | vnodes | cores  |   status   | role  |       create_time       |      offline reason      |
