解决Ubuntu下ufw防火墙与docker端口不受控问题

docker默认使用iptables规则进行流量管理，绕过了ufw的规则。当你使用ufw防火墙时，可能会发现以下情况：

  1. Docker 容器的端口映射（如 -p 8080:80 ）无法被 ufw 的规则控制。
  2. 即使通过 ufw deny 禁用了某个端口，仍然可以访问该端口。
  3. Docker 容器之间的通信可能不受 ufw 的限制。

解决方法：

  1. 通过修改ufw的配置文件，将Docker的流量纳入fw的管理范围
  1. 重启ufw，systemctl restart ufw
  2. 重启docker，systemctl restart docker
  3. 放开端口：ufw route allow proto tcp from any to any port 80
